# 团队协作工作流

自动创建团队、添加合适的成员，并协调执行复杂任务，实现多 Agent 协作完成工作。

## 何时使用

- 需要完成复杂的多步骤任务
- 任务需要不同专业领域的知识
- 用户明确要求团队协作
- 任务涉及研究、规划、实现和审查等多个阶段

## 工作流

1. **任务分析**：分析用户提出的任务，确定所需的专业领域和技能
2. **团队创建**：根据任务需求创建合适的团队
3. **成员配置**：根据任务性质添加不同角色的成员，包括：
   - leader：负责协调和决策
   - worker：负责执行具体任务
   - reviewer：负责审查结果
   - explorer：负责研究和信息收集
4. **任务分配**：根据成员角色分配具体任务
5. **执行协调**：协调团队成员执行任务，确保工作流程顺畅
6. **结果整合**：汇总团队成员的工作结果，向用户呈现最终成果

## 团队配置建议

### 开发类任务
- leader：code 类型，负责整体协调
- worker：code 类型，负责具体实现
- reviewer：test 类型，负责代码审查
- explorer：explore 类型，负责技术研究

### 研究类任务
- leader：explore 类型，负责研究方向
- worker：explore 类型，负责具体研究
- reviewer：explore 类型，负责研究结果审查
- explorer：explore 类型，负责信息收集

### 测试类任务
- leader：test 类型，负责测试策略
- worker：test 类型，负责测试执行
- reviewer：test 类型，负责测试结果分析
- explorer：code 类型，负责代码分析

## 执行策略

- **简单任务**：使用 dispatch 策略，将任务直接分配给最合适的成员
- **复杂任务**：使用 orchestrate 策略，按照 research → plan → implement → review 的流程执行

## 注意事项

- 根据任务复杂度和性质调整团队规模
- 确保团队成员角色覆盖任务所需的所有专业领域
- 定期检查团队执行进度，及时调整策略
- 保持与用户的沟通，确保任务执行符合预期
