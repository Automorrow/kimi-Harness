# Agent 运行诊断工作流

诊断 Kimi Agent 的运行状态和执行问题，帮助定位 Agent 行为异常的根本原因。

## 何时使用

- Agent 执行卡住或超时，需要排查原因
- Agent 输出质量下降或行为不符合预期
- 工具调用失败或返回异常结果
- Agent 执行效率低下，需要分析性能瓶颈
- 需要分析 Agent 的执行日志和决策过程

## 工作流

1. **确认环境状态**：检查 Agent 运行环境，包括版本信息、配置文件、环境变量、网络连接等基础条件
2. **收集日志**：获取 Agent 的执行日志，包括对话历史、工具调用记录、错误信息等
3. **分析执行链路**：梳理 Agent 的完整执行链路，定位异常发生的具体环节（提示词处理、模型调用、工具执行、结果解析等）
4. **检查配置**：验证 Agent 的配置是否正确，包括模型选择、系统提示词、工具定义、权限设置等
5. **隔离问题**：判断问题是出在 Agent 框架层面、模型层面、工具层面还是用户配置层面
6. **复现与验证**：在最小化环境中复现问题，确认诊断结论
7. **给出建议**：根据诊断结果提供具体的修复建议或配置调整方案

## 注意事项

- 区分 Agent 框架问题和模型能力问题，后者通常需要调整提示词或切换模型
- 关注上下文窗口的使用情况，上下文溢出是常见的问题来源
- 工具调用失败时，优先检查工具定义是否清晰、参数是否正确
- 注意 rate limit 和配额限制对 Agent 行为的影响
- 诊断时保留完整的日志和复现步骤，便于后续跟进
